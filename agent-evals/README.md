# Agent Evals for Accessibility Extension

This codebase evaluates the Gemini CLI with the Accessibility Extension installed. It uses [Mocha](https://www.npmjs.com/package/mocha) to run automated tests that interact with the CLI in a pseudo-terminal.

## How it Works

The test pipeline is designed to provide a realistic end-to-end testing environment for the Gemini CLI. Here's a breakdown of the process:

1.  **Environment Setup**: Before the tests run, a global setup hook (`src/mocha/mocha-bootstrap.ts`) is executed. This script, defined in `.mocharc.yml`, calls the `setupEnvironment` function from `src/pipeline/index.ts`.
2.  **Extension Installation**: The `setupEnvironment` function links the local accessibility extension to the Gemini CLI, making it available for the test run. It also prepares any project templates by running `npm install` on them.
3.  **Test Initialization**: Each test file uses the `startAgentTest` function from `src/pipeline/index.ts`. This function:
    *   Creates a unique temporary directory for the test run under `output/`.
    *   Copies a project template (e.g., a starter Next.js app) into the temporary directory.
    *   Spawns a Gemini CLI process in a pseudo-terminal (`node-pty`), controlled by the `GeminiCliRunner`.
4.  **Interaction and Assertions**: The test then interacts with the CLI using the `AgentTestRunner` interface:
    *   `run.type()`: Simulates user input.
    *   `run.expectText()`: Asserts that the CLI output contains specific text.
    *   `run.runCommand()`: Executes shell commands within the test's project directory (e.g., to start a dev server).
    *   `run.expectToolCalls()`: Reads a telemetry log file generated by the CLI to assert that specific tools were called with the expected arguments.
5.  **Cleanup**: After each test, a cleanup hook ensures that the spawned CLI process is terminated.

## Running Tests

The test commands will automatically install the Accessibility Extension before running the test suite.

**WARNING:** Running these evaluations will link the local version of your accessibility extension, which may override any globally installed version.

To run the tests, first install the dependencies:
```bash
# From the agent-evals directory
npm install
```

Then, run the test suite:
```bash
npm run test
```
This command will compile the TypeScript code and then run the Mocha tests.

### Evaluating against the base Gemini CLI

To evaluate against the base Gemini CLI without the accessibility extension, you can use the `test:base` script:

```bash
npm run test:base
```

This is useful for comparing the behavior of the CLI with and without the extension.

## Writing Tests

Add new test files in the `src/tests` directory. The tests follow a pattern of interacting with the Gemini CLI and making assertions about its output and the files it generates. Sample test -

```typescript
import { startAgentTest } from "../pipeline/index.js";
import { AgentTestRunner } from "../pipeline/runner/agent-test-runner.js";
import { promises as fs } from "fs";
import path from "path";
import { expect } from "chai";
import "../mocha/hooks.js";

describe("Accessibility Eval Pipeline", function (this: Mocha.Suite) {
  this.retries(2); // Recommended for tests involving LLMs

  let run: AgentTestRunner;
  let projectDir: string;

  before(async function (this: Mocha.Context) {
    // `startAgentTest` creates a new temporary directory for the test run,
    // and copies the template into it.
    const result = await startAgentTest(this, {
      templateName: "next-app-hello-world", // Specify the template to use
    });
    run = result.runner;
    projectDir = result.projectDir;
  });

  it("should generate an app and run accessibility review", async function (this: Mocha.Context) {
    // 1. Prompt the CLI to generate a web application on top of the template
    await run.type("Create a new login form with a username field, a password field, and a submit button.");
    await run.expectText("Project created successfully!");

    // 2. Start the app's development server (command is fixed due to template)
    const serverOutput = await run.runCommand("npm run dev", /ready started server on/i);
    const urlMatch = serverOutput.match(/(http:\/\/localhost:\d+)/);
    const url = urlMatch![0];

    // 3. Run the accessibility review command
    await run.type(`/accessibility:review ${url}`);
    await run.expectToolCalls(["a11y_audit_web_url"]);
    await run.expectText("Accessibility review complete.");

    // 4. Assert that the report was created
    const reportPath = path.join(projectDir, "ACCESSIBILITY_REVIEW_TODO.md");
    const reportContent = await fs.readFile(reportPath, "utf-8");
    expect(reportContent).to.include("Accessibility Violations");
  });
});
```
